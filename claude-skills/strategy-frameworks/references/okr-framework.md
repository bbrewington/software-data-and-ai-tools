# OKR Framework Reference

Detailed guide to writing and managing OKRs.

## Overview

OKRs (Objectives and Key Results) is a goal-setting framework that links ambitious objectives to measurable outcomes. Popularized by Intel and Google, OKRs create alignment and focus across organizations.

## Writing Good Objectives

### Characteristics

- Qualitative and inspirational
- Action-oriented (verb + outcome)
- Time-bound (quarterly)
- Ambitious but achievable
- Aligned with strategy

### Examples

| Bad | Good |
|-----|------|
| "Improve customer satisfaction" | "Become the most loved product in our category" |
| "Increase revenue" | "Achieve breakthrough growth in enterprise" |
| "Build features" | "Transform how teams collaborate" |

## Writing Good Key Results

### Characteristics

- Quantitative and measurable
- Outcome-focused (not tasks)
- 3-5 per objective
- Specific and unambiguous
- Ambitious (stretch goals)

### Examples

| Bad | Good |
|-----|------|
| "Launch feature X" | "50% of users adopt new workflow" |
| "Meet with customers" | "Increase NPS from 30 to 50" |
| "Improve quality" | "Reduce P1 bugs by 60%" |

## OKR Types

| Type | Achievement Target | Use For |
|------|-------------------|---------|
| Committed | 100% | Business requirements |
| Aspirational | 70% | Stretch goals |

## Cadence

| Activity | Frequency |
|----------|-----------|
| Set OKRs | Quarterly |
| Check-ins | Weekly |
| Scoring | End of quarter |
| Review | Quarterly |

## Scoring

| Score | Meaning |
|-------|---------|
| 0.0-0.3 | Failed to progress |
| 0.4-0.6 | Progress but short |
| 0.7-1.0 | Delivered (0.7 = success for aspirational) |

## Common Mistakes

| Mistake | Solution |
|---------|----------|
| Too many OKRs | 3-5 objectives max |
| KRs as tasks | Focus on outcomes |
| Sandbagging | Embrace stretch |
| Set and forget | Weekly check-ins |
| Punishing misses | Celebrate learning |
