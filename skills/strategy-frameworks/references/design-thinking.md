# Design Thinking / Double Diamond Reference

Detailed methodology for human-centered design.

## Overview

Design Thinking is a human-centered approach to innovation that integrates user needs, technology possibilities, and business requirements. The Double Diamond model visualizes the process as alternating phases of divergent and convergent thinking.

## The Four Phases

### 1. Discover (Diverge)

**Goal**: Understand the problem space

| Activity | Description |
|----------|-------------|
| User research | Interviews, observation, immersion |
| Stakeholder interviews | Understand constraints, goals |
| Competitive analysis | What exists today |
| Data analysis | Quantitative insights |

**Output**: Rich understanding of users and context

### 2. Define (Converge)

**Goal**: Focus on the right problem

| Activity | Description |
|----------|-------------|
| Synthesis | Pattern finding, affinity mapping |
| Personas | Archetypal users |
| Journey maps | Experience visualization |
| Problem framing | "How might we..." statements |

**Output**: Clear problem definition

### 3. Develop (Diverge)

**Goal**: Generate possible solutions

| Activity | Description |
|----------|-------------|
| Ideation | Brainstorming, sketching |
| Concept development | Rough solutions |
| Prototyping | Quick, low-fidelity tests |
| User testing | Feedback on concepts |

**Output**: Range of potential solutions

### 4. Deliver (Converge)

**Goal**: Implement the solution

| Activity | Description |
|----------|-------------|
| Refinement | Iterate based on feedback |
| Final design | High-fidelity solution |
| Testing | Validation with users |
| Launch | Release and measure |

**Output**: Working solution

## Key Methods

### Discovery Methods

| Method | Description | When to Use |
|--------|-------------|-------------|
| User interviews | 1:1 conversations | Deep understanding |
| Contextual inquiry | Observe in context | Real behavior |
| Diary studies | Self-reported over time | Longitudinal insights |
| Surveys | Quantitative questions | Scale validation |

### Definition Methods

| Method | Description | Output |
|--------|-------------|--------|
| Affinity mapping | Group insights by theme | Patterns |
| Personas | Archetypal user profiles | Focus |
| Journey maps | Visualize experience | Pain points |
| HMW statements | Reframe as opportunity | Direction |

### Development Methods

| Method | Description | Fidelity |
|--------|-------------|----------|
| Brainstorming | Rapid idea generation | Concept |
| Sketching | Quick visualization | Low |
| Paper prototypes | Physical mockups | Low |
| Wireframes | Digital structure | Medium |
| Clickable prototypes | Interactive mockups | High |

### Delivery Methods

| Method | Description | Purpose |
|--------|-------------|---------|
| Usability testing | Watch users try it | Validation |
| A/B testing | Compare variations | Optimization |
| Pilot programs | Limited release | Learn before scale |
| Metrics tracking | Measure outcomes | Success criteria |

## How Might We (HMW)

Transform insights into actionable opportunities:

| Insight | HMW Statement |
|---------|---------------|
| "Users forget to..." | "HMW make it easy to remember?" |
| "The process is slow" | "HMW reduce time by half?" |
| "Users feel anxious" | "HMW create confidence?" |

## Common Mistakes

| Mistake | Problem | Solution |
|---------|---------|----------|
| Skipping discovery | Wrong problem | Invest in research |
| Falling in love with first idea | Missed opportunities | Generate many ideas |
| Not testing with users | Assumptions untested | Prototype and test |
| Linear process | Missing iterations | Loop back as needed |
| Design theater | No real change | Commit to implementation |

## Design Thinking vs. Lean

| Aspect | Design Thinking | Lean Startup |
|--------|-----------------|--------------|
| Focus | Desirability first | Viability first |
| Method | Deep research | Rapid experiments |
| Prototype | Test understanding | Test assumptions |
| Best for | Novel problems | Business model validation |
