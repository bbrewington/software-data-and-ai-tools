# Customer Maturity Model Reference

Detailed methodology for designing customer maturity models.

## Overview

A customer maturity model describes how customer sophistication, needs, or practices evolve over time. It helps segment customers, design tiered offerings, and plan customer success strategies.

## Building a Maturity Model

### Step 1: Define Dimensions

What aspects of maturity matter?
- **Capability**: What can they do?
- **Process**: How mature are their processes?
- **Technology**: What tools do they use?
- **Organization**: How sophisticated is their org?

### Step 2: Define Levels

Typically 4-5 levels from beginning to leading:

| Level | Name | Description |
|-------|------|-------------|
| 1 | Initial / Ad-hoc | Just starting, reactive |
| 2 | Developing | Building capability |
| 3 | Defined | Established practices |
| 4 | Managed | Measured and controlled |
| 5 | Optimizing | Continuously improving |

### Step 3: Define Characteristics

For each level, document:
- What they look like
- What they can do
- What they struggle with
- What they need

### Step 4: Map to Offerings

Connect levels to your products/services.

## Applications

| Use Case | How Model Helps |
|----------|-----------------|
| Segmentation | Target marketing by level |
| Product tiers | Match features to maturity |
| Pricing | Value-based by sophistication |
| Success | Define progression paths |
| Sales | Qualify and position |
